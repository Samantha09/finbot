# ETF 自适应网格交易策略工具设计

## 概述

新增 `etfGridTrading` 工具，对指定 ETF 基金生成一周期的自适应网格交易策略。结合广发证券 API（基本面 + 资金面）和东方财富 K 线数据（技术面），自动计算网格上下界、网格密度、交易计划、风控建议。

## 输入参数

```typescript
{
  symbol: string;              // ETF 代码，如 "510050.SH"、"159915.SZ"
  holdings: {                  // 用户当前持仓
    avgPrice: number;          // 持仓均价
    quantity: number;          // 持仓数量（股）
  };
  capital?: number;            // 可用资金（可选）
  period?: "daily" | "weekly"; // K 线周期，默认 daily
}
```

## 数据获取

| 步骤 | 数据源 | 获取内容 | 用途 |
|------|--------|----------|------|
| 1 | 广发 `gfFundDetail` | ETF 规模、近期涨跌幅、申购赎回状态 | 基本面筛选 + 风控参考 |
| 2 | 广发 `gfEtfSuperFund` | 近期资金流入/流出 | 判断资金面强弱 |
| 3 | 东方财富 `fetchKlines` | 120 日 K 线（OHLCV） | 技术分析 + 网格边界计算 |

复用 `technical-analysis.ts` 的 `calcMA`、`calcBOLL`、`calcRSI`、`calcMACD` 函数，不重新造轮子。新增 ATR 计算函数放在本工具内。

## 自适应网格计算

### 网格边界

- **上界**：`min(布林带上轨, 近 60 日最高价 × 1.02)`
- **下界**：`max(布林带下轨, 近 60 日最低价 × 0.98)`
- 边界确定后标注依据（布林带/前高前低）

### 网格密度

- 基于 ATR（Average True Range，14 日）动态划分
- 每格宽度 ≈ 0.5~1 倍 ATR
- 关键支撑/阻力位附近加密（RSI 超买超卖区间、MA 交叉点）
- 总网格数控制在 8~20 格之间
- 网格间距非均匀分布

### 持仓映射

- 将用户持仓均价映射到网格中
- 标记当前所在格子
- 区分持仓格上方（需卖出）和下方（可买入）

## 输出格式

### 1. 网格参数表

| 字段 | 说明 |
|------|------|
| 网格上界/下界 | 自适应计算结果 + 依据 |
| 网格数 | 8~20 格 |
| 每格价格 | 非均匀分布，关键位附近密集 |
| 网格间距 | 每格价差和百分比 |
| 当前持仓位置 | 持仓均价在网格中的位置 |

### 2. 交易计划表

每格一行：格子编号 | 买入价 | 卖出价 | 建议股数 | 预期单格收益 | 资金占用

传入 capital 时额外输出：总资金占用比例、资金利用率、每格资金分配。

### 3. 风控建议

- **破网处理**：价格突破上界/下界时的操作建议
- **风控指标**：单格最大亏损、最大持仓占比、资金使用率
- **止盈止损**：基于 RSI + 布林带的建议止盈/止损点

### 4. 技术分析依据

简要列出用于确定网格边界的指标值：布林带三轨、ATR、MA 排列、RSI、资金流向。

### 5. 结构化数据

底部附加 JSON 格式的网格数据，方便其他工具或回测系统消费。包含完整的网格参数和交易计划。

## 错误处理

- K 线数据不足（< 60 条）：返回 `isError: true`，提示数据不足
- 广发 API 失败：降级为仅技术分析模式，标注"基本面数据不可用"
- 无效 symbol：通过 `fetchKlines` 已有的验证逻辑捕获
- 所有错误走 `toToolResult({ content, isError: true })`

## 不做的事情

- 不做实时监控/自动下单（只是策略规划工具）
- 不做回测（已有 `strategy-backtest.ts`）
- 不修改 `technical-analysis.ts`（只复用导出函数）
- 不引入新的 npm 依赖
